# TiebaPure-iOS 隐私说明

更新日期：2026-07-13。

TiebaPure-iOS 是非官方、开源的百度贴吧只读客户端。本项目自身不包含广告 SDK、分析 SDK、用户画像、跨应用跟踪或自建遥测服务器。

## 本机保存的数据

- 可选登录账号：仅保存通过验证的 UID、昵称、头像标识、TBS，以及最小的 `BDUSS`、`STOKEN`、`BAIDUID`，存入 iOS Keychain，访问级别为 `WhenUnlockedThisDeviceOnly`。
- 最近浏览贴吧：最多 30 条吧名、显示名、头像 URL 和更新时间，保存在 UserDefaults。用途属于应用自身功能，对应隐私清单理由 `CA92.1`。
- 图片缓存：公开帖子中的图片可能由独立的 URL 缓存保存在本机，以改善阅读性能；应用内“退出登录”会同时清除内存和磁盘图片缓存，卸载应用会删除应用容器。
- 用户主动选择“保存原图”时，应用仅请求向系统照片库添加该图片的权限，不读取或枚举现有照片。下载仍受 HTTPS、图片 MIME 和 30 MiB 上限约束。

应用不新增帖子阅读历史。

## 登录

登录页由百度 HTTPS 页面提供，使用非持久 `WKWebsiteDataStore`。应用只允许百度域边界内的 HTTPS 导航，只在精确的贴吧成功页面读取 Cookie，并只接受名称白名单、Secure、未过期且属于百度域的三类必要 Cookie。应用不会保存或转发浏览器的完整 Cookie 字符串。

退出登录会清理旧 WebKit 百度数据、`HTTPCookieStorage`、URL/图片缓存和 Keychain。只有所有清理全部成功后才发布“已退出”状态；任一步失败时保持已登录并向用户显示错误。

## 网络请求

应用直接与百度贴吧及内容 CDN 的 HTTPS 服务通信。网络运营商、系统 DNS 和百度服务可能按其自身政策处理 IP 地址、请求时间、设备网络信息和用户请求的数据。

API 使用不持久化的 ephemeral URLSession，不自动保存 Cookie、URL 凭证或敏感响应缓存。API 响应限制为 16 MiB，图片限制为 30 MiB；图片还必须返回 `image/*` MIME。

## 数据共享

本项目没有自建后端，因此不会把账号或阅读行为上传给项目维护者。用户主动打开网页、分享链接或使用百度登录时，数据由所选系统功能或百度服务按其政策处理。

## 用户控制

- “退出登录”清除本机登录状态。
- 删除应用会清除应用容器中的 UserDefaults 和缓存；iOS 不保证卸载时删除 Keychain 项。如需确定删除凭证，请先在应用内执行“退出登录”。
- 最近浏览贴吧目前没有单独同步到云端。
